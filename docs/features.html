Capabilities

Features

25 skills that turn Claude Code into a knowledge engine. Every operation is a slash command.

Ingest

Process any source into wiki pages

Drop in a URL, PDF, YouTube video, tweet, gist, Office doc, or pasted text. Claude reads it, extracts key information, creates source notes, entity pages, and concept pages. Updates the index, appends to the log, and auto-commits.

A single source typically creates 3-7 wiki pages. All with YAML frontmatter, wikilinks, and source provenance.

7 source types. Dependencies auto-install on first use.

ingest
> /ingest https://article.com --vault research Reading source... Creating pages: + wiki/sources/article-name.md + wiki/entities/author-name.md + wiki/concepts/key-idea.md ~ wiki/index.md (updated) ~ log.md (appended) Committed: ingest article-name
Query

Ask questions, get cited answers

Claude reads your wiki's index to find relevant pages, reads them, and synthesizes a comprehensive answer with citations to specific wiki pages.

The key insight: good answers can be filed back into the wiki. Add --save and the answer becomes a new wiki page. Your explorations compound just like ingested sources.

query
> /query "Compare RAG vs wiki" --save Reading index... found 4 relevant pages Synthesizing answer... RAG re-derives each query. A persistent wiki compiles once, keeps current. See [[rag-vs-persistent-wiki]] Answer saved to: + wiki/comparisons/rag-vs-wiki.md
Lint

Health-check your knowledge

8 automated checks: orphan pages, broken wikilinks, missing frontmatter, empty source links, missing domain tags, concepts without pages, stale content, index completeness.

Reports are grouped by severity. Add --fix to auto-repair what's fixable — missing frontmatter, stub pages for broken links, domain tags from vault defaults.

lint
> /lint --vault research --fix Scanning 42 pages... Critical: x [[missing-page]] referenced but missing Warning: ! old-tool.md — orphan (0 inlinks) ! some-idea.md — no domain tag Info: i "machine learning" — 8 mentions, no page Auto-fixed: 2 issues
Promote

Graduate knowledge between vaults

When finishing a project, promote reusable learnings to your meta vault. Tech patterns, strategy insights, vendor evaluations — anything useful across projects.

Claude reads the project vault, identifies what's reusable vs project-specific, creates or merges pages in the target vault, and updates both indexes. A promoted-from frontmatter field tracks provenance.

promote
> /promote project-alpha --to meta Scanning project-alpha vault... Identified 5 promotable pages: Promoted: + meta/concepts/nextjs-deploy.md + meta/entities/vercel.md ~ meta/concepts/auth-patterns.md (merged) Skipped: - client-api-schema.md (project-specific)
Search

Full-text search at scale

At small scale (~100 pages), the index.md catalog is all you need. When your wiki grows, add qmd for hybrid BM25 + vector search with LLM re-ranking. All local, no API keys.

Falls back to grep-based search if qmd isn't installed. Can also run as an MCP server for native tool access.

search
> /search "deployment patterns" --vault meta Results (qmd hybrid search): 1. [[nextjs-deploy]] concept ...ISR, SSR, and edge deployment... 2. [[vercel]] entity ...Fluid Compute, auto-scaling... 3. [[docker-patterns]] concept ...multi-stage builds, Watchtower...
Slides

Presentations from your wiki

Generate Marp-format slide decks from wiki content. Claude reads relevant pages, synthesizes a focused presentation, and saves it as a wiki page you can view in Obsidian or export to PDF/HTML.

Slides are wiki pages too — they get frontmatter, appear in the index, and compound.

slides
> /slides "LLM Knowledge Bases" --vault research Reading 6 relevant pages... Generating 12-slide deck... + wiki/slides-llm-knowledge-bases.md View in Obsidian (Marp plugin) or export: npx @marp-team/marp-cli slides.md --pdf
