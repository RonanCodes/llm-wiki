LLM Wiki

A personal knowledge base powered by LLMs. Claude Code builds your wiki. Obsidian is where you read it.

Inspired by Karpathy's LLM Wiki pattern
You (terminal) You (Obsidian) | | |-- /ingest <source> |-- Browse wiki pages |-- /query "question" |-- Graph view |-- /lint |-- Backlinks |-- /promote |-- Search | | '-- Claude Code --writes--> vaults/<name>/wiki/*.md

Quick Install

On macOS, run the install script which handles everything:

git clone https://github.com/RonanCodes/llm-wiki.git
cd llm-wiki
./install.sh

The script will check for each dependency and offer to install anything missing. It uses Homebrew under the hood.

What You Need

Claude Code required

The AI engine that powers everything. Reads sources, writes wiki pages, answers questions.

npm i -g @anthropic-ai/claude-code

Documentation →

Obsidian required

Your wiki viewer. Browse pages, see the graph view, follow backlinks, search.

brew install --cask obsidian

Download →

Git required

Version control for the engine and every vault. Each vault is its own git repo.

brew install git

Node.js required

Runtime for Claude Code and npm packages.

brew install node

Download →

yt-dlp auto-installed

YouTube transcript extraction. Installed automatically when you first ingest a YouTube video.

brew install yt-dlp

Poppler auto-installed

PDF text extraction. Installed automatically when you first ingest a PDF.

brew install poppler

Pandoc auto-installed

Word, Excel, PowerPoint conversion. Installed automatically on first use.

brew install pandoc

Obsidian Web Clipper optional

Browser extension to save web articles as markdown directly into your vault.

Install extension →

Getting Started

Run the installer

./install.sh

Checks your environment and installs anything missing. Interactive — asks before installing each tool.

Open Claude Code

claude

Launch Claude Code in the llm-wiki directory. It reads CLAUDE.md and knows all the available skills.

Create your first vault

/vault-create my-research --domain ai-research

Creates a vault at vaults/my-research/ with the full wiki structure and initializes it as a git repo.

Open in Obsidian

Open Obsidian → Open folder as vault → select vaults/my-research. You'll see the wiki structure with index.md, log.md, and the wiki subdirectories.

Ingest your first source

/ingest https://some-article.com --vault my-research

Claude reads the article, creates wiki pages (source summary, entities, concepts), updates the index, and auto-commits. Watch the pages appear in Obsidian in real time.

Ask questions

/query "What are the key takeaways?" --vault my-research

Claude reads the wiki, synthesizes an answer with citations. Add --save to file the answer back into the wiki.

Commands

CommandWhat it does
/vault-create <name>Create a new vault with full wiki structure
/vault-import <path>Import an existing Obsidian vault or markdown folder
/vault-statusShow all vaults with page counts and status
/ingest <source>Process any source into wiki pages (URL, PDF, YouTube, tweet, etc.)
/query "question"Ask questions against the wiki, get answers with citations
/search "terms"Full-text search across wiki pages
/lintHealth-check the wiki for issues (orphans, broken links, etc.)
/promote <vault>Graduate reusable knowledge to another vault
/slides "topic"Generate a Marp presentation from wiki content
/setupFirst-time machine setup (check/install dependencies)
