## Codebase Patterns
- Skills use SKILL.md format inside named directories under .claude/skills/
- YAML frontmatter fields: name, description, argument-hint, disable-model-invocation, allowed-tools, user-invocable
- Vault CLAUDE.md files are thin config (conventions, not logic)
- Use Obsidian-compatible wikilinks [[page-name]] for cross-references
- Every wiki page must have frontmatter AND link back to raw sources
- Domain tags inherited from vault default, can be extended per-page
- Commit messages use emoji conventional commit format
