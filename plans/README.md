# Plans

Design plans generated by `/write-a-prd --plan`. These are human-readable planning artifacts — review them, adjust them, then convert to `.ralph/prd.json` for execution.

## How it works

```
/write-a-prd "feature name"

  Choose a mode:
  ┌─────────────────────────────────┬──────────────────────────────────┐
  │  --quick                        │  --plan                          │
  │                                 │                                  │
  │  Interview                      │  Interview                       │
  │       ↓                         │       ↓                          │
  │  Generate prd.json              │  Generate plans/feature.md       │
  │       ↓                         │       ↓                          │
  │  Ready for /ralph               │  You review & adjust             │
  │                                 │       ↓                          │
  │                                 │  "Looks good" → prd.json         │
  │                                 │       ↓                          │
  │                                 │  Ready for /ralph                │
  └─────────────────────────────────┴──────────────────────────────────┘
```

## What's in a plan

Each plan includes:

- **Architectural decisions** — durable choices (routes, schema, models) locked in before building
- **Vertical slices** — tracer-bullet phases, each cutting through all layers end-to-end
- **Acceptance criteria** — checkboxes you can tick off as phases complete

## File layout

```
plans/
├── README.md          ← this file
├── search-feature.md  ← example plan
└── auth-rework.md     ← another plan
```

Plans are committed to the repo. Execution state (`.ralph/prd.json`, `.ralph/progress.txt`) is gitignored.
