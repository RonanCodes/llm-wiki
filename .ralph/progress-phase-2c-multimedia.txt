## 2026-04-18 11:52 - US-001: Implement generate-podcast handler

- Created `.claude/skills/generate-podcast/SKILL.md` — 10-step pipeline matching the Phase 2A/2B cadence (deps → resolve → hash → script → TTS → concat → sidecar → commit → report).
- TTS backend priority: ElevenLabs (if `ELEVENLABS_API_KEY`) → OpenAI TTS (if `OPENAI_API_KEY`) → Piper (local, free, lazy-installed). Documented the quality/cost tradeoff in a table.
- Two templates at `templates/single-host.md` and `templates/two-voice.md` — monologue and NotebookLM-style dialogue. Placeholder-driven; the invoking LLM fills them using wiki content.
- Script conventions documented: ~150 wpm, `{{cite: path}}` preprocessed to `*pagename*`, sources spoken-list at the end.
- ffmpeg VBR q2 MP3 concat — right quality for voice, smaller than CBR without audible loss. 250ms inter-line silence, 600ms on speaker change.
- `.script.md` kept alongside `.mp3` — it's diffable, re-renderable, and the honest primary artifact. The MP3 is derived.
- Reuses `.claude/skills/generate/lib/select-pages.sh` and `source-hash.sh` — no duplication.
- **Learnings for future iterations:**
  - Cost guardrail noted in Known Limitations: for long ElevenLabs podcasts, print projected cost before rendering and confirm. Generate-video will inherit this when it chains podcast for voiceover.
  - Piper voices: en_US-lessac-medium + en_GB-alan-medium for the two-voice default. Robotic but serviceable — the right "draft listen" tier. Good users will set an OpenAI/ElevenLabs key and get the upgrade automatically.
  - `.script.md` as a first-class artifact is the right pattern — it gives PRs something diffable, avoids the "binary black box" problem, and makes re-rendering free after editing.
  - Two templates felt right (monologue + dialogue). Resist the urge to ship a third ("interview with expert", etc.) until someone actually asks — templates rot without feedback.
  - Handler is script-shape-agnostic: the parser just splits on `[HOST]` / `[A]` / `[B]`. New formats slot in by adding a new template + documenting the tags.

## 2026-04-18 11:58 - US-002: Implement generate-video handler

- Created `.claude/skills/generate-video/SKILL.md` — 9-step pipeline (deps → resolve+hash → scenes.json → pick composition → remotion render → optional voiceover mux → sidecar → commit → report).
- `REMOTION_STUDIO_DIR` env var (default `~/Dev/ai-projects/remotion-studio`) locates the external renderer — the handler treats remotion-studio as a peer repo, not a submodule. Works for CI + container overrides.
- Composition picker enumerates real existing compositions I read from `src/Root.tsx`: KineticPitch (32s), PromoV10 (56s), PromoV2–V9 (style variants, light/dark/synth/screenshots), AppDemo, MarketingPromo. Listed `wiki-explainer` as to-be-added default (fallback to PromoV10 until it lands).
- scenes.json contract documented — `title`, `subtitle`, `voiceover_script`, `scenes[]` with `id`, `duration_s`, `props{}`. Remotion's `--props=<file>` flag takes this directly; Remotion components destructure from input props.
- `--voiceover` chains `generate-podcast` — produces MP3, then `ffmpeg -map` muxes into the silent Remotion render. `-shortest` trim guards mismatched durations.
- Sidecar pins `remotion-studio-commit` — essential for re-renderability because composition props can drift between commits of the external repo.
- scenes.json kept alongside MP4 as the re-renderable source (same pattern as `.script.md` for podcasts and `.outline.md` for mindmaps).
- **Learnings for future iterations:**
  - Treating remotion-studio as a peer repo (not vendored, not submodule) is the right call for now — it's Ronan's actively-iterated space. The `remotion-studio-commit` sidecar pin is sufficient provenance.
  - Handler scope intentionally stops short of building a `wiki-explainer` composition — that's a React/Remotion artifact, belongs in remotion-studio, not here. Documented as a follow-up in Known Limitations.
  - Composition durations are hardcoded in Root.tsx (`durationInFrames`). Voiceover must match or `-shortest` will truncate one or the other. Phase 2E should add a duration check before rendering.
  - The auto-pick rule uses "vs"/"versus" and scene shape — same heuristic as generate-infographic. Any future comparison-themed handler should hook into this same signal. Candidate for a shared `detect-shape.sh` helper if a 4th user emerges.
  - `render.sh` for generate-podcast is referenced but optional — the invoking LLM can also call `/generate podcast` and capture the MP3 path from the report. Simpler, same effect. Real shell helper lands when a non-LLM caller needs it.
