## 2026-04-18 14:49 - US-002: Implement /verify-artifact skill (round-trip fidelity test)

- Wrote `.claude/skills/verify-artifact/SKILL.md` — first user-invocable skill alongside `/generate`. 8-step pipeline: select source pages → run `/generate` → scratch vault under `/tmp` → pick inverse path → re-ingest → diff → score → cleanup.
- **Inverse-path choice (the key design call):** for each artifact type, prefer the re-renderable source sidecar (`.script.md`, `.scenes.json`, `.questions.json`, `.cards.csv`, `src/data.json`) over the binary form (PDF/MP3/MP4/SVG/HTML). Re-renderable sidecars are cheap, deterministic, and avoid a hard whisper/OCR dependency for the common case. The binary paths (whisper for MP3, whisper+OCR for MP4, pdftotext for PDF) are documented but deferred to future iterations — today they'd require gating by `--binary-ingest` and a whisper install check.
- **Scoring formula:** weighted `fidelity = 0.6*coverage + 0.4*jaccard`. Coverage = fraction of original pages whose re-ingested counterpart retained ≥50% of the original's concept set. Jaccard = corpus-wide concept set similarity. The 60/40 split rewards "every page survived" over "vocabulary overlapped" — a lossy artifact that preserves 3/10 pages perfectly should score lower than one that preserves 10/10 pages loosely, and the split achieves that.
- **Concept extraction is heuristic:** wikilink targets (`[[page-name]]`), frontmatter `tags:`, and capitalised multi-word terms. Both false positives and negatives exist — e.g. a well-structured page with many bare terms will under-extract. Documented this as a known limitation. The LLM-judge tier (`--llm-judge`) is the antidote but is flag-gated because it's the expensive path.
- **Target table encoded from close-the-loop-testing.md:** book 0.85, pdf 0.85, podcast 0.75, video 0.60, mindmap 0.50, flashcards 0.40, quiz 0.40, slides 0.35, app 0.25, infographic 0.25. Per-type defaults; `--target` overrides.
- **CI story:** exit 0 = pass, 1 = fidelity below target, 2+ = infra error (missing artifact, ingest handler failure, etc.). Golden-corpus workflow in US-004 will chain this.
- **Flags shipped:** `--target <float>`, `--llm-judge`, `--keep-scratch` (preserves `/tmp/verify-<random>/` for manual inspection), `--from <path>` (verify an existing artifact without regenerating — useful after a git pull when you want to confirm nothing drifted).
- **Learnings for future iterations:**
  - Designing the *inverse* was harder than designing the *forward*. `/generate` was "pick format, render"; `/verify-artifact` is "which of N paths back to markdown is least lossy and cheapest?" — and the answer differs per artifact type. Writing out the inverse-path preference table first (in the skill's Mermaid + its own paragraph) made the Python scoring trivial afterwards.
  - Weighting concept-level metrics (coverage + jaccard) was an intentional choice over line-level metrics (diff %, edit distance). Line-level would reward the book format (which reproduces text verbatim) and punish the slides format (which rewords heavily) — but slides are *allowed* to reword. Concept-level rewards "did the ideas survive" regardless of surface form.
  - Flag-gate the expensive tier (`--llm-judge`) rather than running it always. A fidelity check should be fast enough that CI can run it per-push; LLM judging would balloon runtime and cost without being load-bearing for the happy path.
  - The `--from` flag came from working through the drift-detection story (US-003) in my head while writing this skill — they're complementary. `/lint --artifacts` flags drift cheaply; `/verify-artifact --from <path>` re-verifies the flagged ones without re-generating. Shipping both means the loop can open from either end.
  - Python heredoc inside the bash skill works fine for the scoring math, but it's the longest inline Python in the repo so far. If a future handler needs richer scoring (e.g. embedding similarity), extract to `.claude/skills/verify-artifact/lib/score.py`.
